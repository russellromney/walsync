## 1. Install Walsync

```bash
cargo install walsync
```

## 2. Set Up S3 Credentials

Create a `.env` file or export environment variables:

```bash
export AWS_ACCESS_KEY_ID=your_key
export AWS_SECRET_ACCESS_KEY=your_secret
export AWS_ENDPOINT_URL_S3=https://fly.storage.tigris.dev
export AWS_REGION=auto
```

## 3. Take Your First Snapshot

```bash
walsync snapshot /path/to/app.db -b s3://my-bucket
```

You'll see:
```
Snapshot uploaded: s3://my-bucket/app/snapshots/20240112120000.db
```

## 4. Watch for Changes

Keep Walsync running to continuously sync WAL changes:

```bash
walsync watch /path/to/app.db -b s3://my-bucket
```

The process will:
- Monitor your database for changes
- Upload new WAL frames to S3 automatically
- Take periodic snapshots (default: hourly)

## 5. List Your Backups

See what's backed up:

```bash
walsync list -b s3://my-bucket
```

Output:
```
  app (5 snapshots, 12 WAL segments)
```

## 6. Restore a Database

Restore the latest snapshot:

```bash
walsync restore app -o recovered.db -b s3://my-bucket
```

Or restore to a specific point in time:

```bash
walsync restore app -o recovered.db -b s3://my-bucket --point-in-time 2024-01-12T12:00:00Z
```

## Python API

Integrate Walsync into your Python application:

```python
from walsync import WalSync

# Create an instance
ws = WalSync("s3://my-bucket", endpoint="https://fly.storage.tigris.dev")

# Snapshot your database
ws.snapshot("/path/to/app.db")

# List backed-up databases
databases = ws.list()
print(f"Backed up: {databases}")

# Restore a database
ws.restore("app", "/path/to/recovered.db")
```

## Multi-Database Setup

Walsync shines with multiple databases - single process handles them all:

```bash
walsync watch \
  /var/lib/app/tenant1.db \
  /var/lib/app/tenant2.db \
  /var/lib/app/tenant3.db \
  -b s3://backups/app
```

This saves **200MB+ memory** vs running separate Litestream processes.

## What's Happening?

```
Your Database          S3 Backup
┌─────────────┐       ┌─────────────────────┐
│  app.db     │       │ app/snapshots/...   │  ← Full snapshots
│             │──────→│ app/wal/...         │  ← WAL segments
│  app.db-wal │ watch │ app/state.json      │  ← Checkpoint state
└─────────────┘       └─────────────────────┘
```

1. **Snapshot** - Full database backup (taken hourly)
2. **WAL Segments** - Incremental changes between snapshots
3. **State** - Checkpoint information for recovery

## Troubleshooting

### "WALSYNC_TEST_BUCKET not set"
This is only for running tests. For normal use, just specify `-b s3://bucket`.

### "Connection refused"
Check that S3 credentials and endpoint are correct:
```bash
echo "Key: $AWS_ACCESS_KEY_ID"
echo "Endpoint: $AWS_ENDPOINT_URL_S3"
```

### "No snapshots found"
This is normal on first run. Create a snapshot:
```bash
walsync snapshot /path/to/app.db -b s3://my-bucket
```
